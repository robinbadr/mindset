WordPress
Theme Types

Block vs Classic

There are two main types of themes in WordPress: block and classic.

Theme Handbook: Theme Types

Block Themes

A block theme uses an HTML-based templating system. The entire site is built using the block editor and custom blocks can be created using React.

Twenty Twenty-Four is an example of a block theme.

Classic Themes

A classic theme uses a PHP-based templating system. The site can use the block editor, the classic editor, or neither.

Twenty Twenty is an example of a classic theme.

Hybrid Themes

A classic theme that opts in to some features of the block editor is unofficially called a hybrid theme.

Twenty Twenty One is an example of a hybrid theme.

Which theme type will we build for the Mindset site?

Block Themes Classic/Hybrid Themes
More client control More developer control
Faster to build Slower to build
Older WP devs haven't learned it Most WP sites still use them
New features being added regularly Increasingly becoming legacy code

Building a WordPress Site

Choosing a Theme

You can build a WordPress website multiple ways, regardless of the theme type (block or classic).

First decide what theme you are going to use or build:

  • Pre-built theme
  • Child theme
  • Custom theme

Pre-built Themes

Free Themes

  • Choose themes on the WordPress theme repository.
  • Check reviews and number of installations.
  • Check when it was last updated and if it has support.

Paid Themes

  • Paid (Premium) themes are inexpensive and useful if only a few tweaks are needed (Ex: color, font size.)
  • Purchase themes from reputable sellers (Ex: ThemeForest, StudioPress, CSSIgnitor).
  • Check reviews and number of installations.
  • Check when it was last updated and if it has support.

Page Builders

Many paid themes use plugins that replace the block editor with a custom page builder.

Examples: Elementor, Divi, WP Bakery, Breakdance.

If you build a WordPress site with a page builder, you can never uninstall it!

Parent & Child Themes

Child Themes

A WordPress child theme is a theme that inherits the functionality of another theme, called the parent theme, and allows you to modify, or add to the functionality of that parent theme.
--WordPress Codex

Child Themes

  • Use it when you want to use an existing theme but change some styles, functions, templates.
  • Your customization is preserved but you can still get updates to the parent theme.

Custom-built Themes

Custom-built Themes

  • Perfect choice when you need full control.
  • You use a starter theme always, never start from scratch.
  • The starter theme has the base files. You add your custom code according to each project.

Starter Themes

Where you work and what you work on will impact which starter theme you use. There are many out there.

  • _s by Automattic
  • _tw by Greg Sullivan
  • Create Block Theme by WordPress.org
  • Blockbase by Automattic

Themes Recap

  • Block themes are used to build WordPress sites entirely within the block and site editor.
  • Classic themes are PHP-based and allow developers to decide how content is stored and displayed on a WordPress site.
  • Block, classic, and hybrid themes can all be either pre-built, parent/child, or custom-built.

theme.json - version

You should use the latest version available of the API which is currently version 3.

theme.json Version 3 Reference

theme.json - schema

The schema is a reference document to provide error checking, tooltips, and autocomplete in your code editor.

The latest version can always be found at
https://schemas.wp.org/trunk/theme.json
but it assumes you are using the development plugin with beta features.

theme.json - settings

The settings section allows you to enable, disable, and configure features of the Site Editor.

For example, you can enable or disable the ability to set margin or padding values on blocks within the Editor. This can be done for all blocks or for specific blocks.

theme.json Settings - Theme Handbook

theme.json - styles

The styles section allows you to specify styles (CSS) at a global level, a block level, or an HTML element level.

For example, you can set a default font color, a different color for <a> elements, and another color for the Pullquote block.

theme.json Styles - Theme Handbook

theme.json - customTemplates

If you create custom template files in your theme that you want users to be able to use on any Post or Page, you can use this section to register them.

theme.json Custom Templates - Theme Handbook

theme.json - templateParts

If you create template part files in your theme like footer.html or header.html, you can use this section to register them.

theme.json Template Parts - Theme Handbook

theme.json - patterns

The patterns section is for specifying patterns from the Pattern Directory to include in your theme.

This is not used for patterns you create and put in your patterns folder because those load automatically.

theme.json Patterns - Theme Handbook

Templates

CMS Templates

"In a nutshell, templates take dynamic data and wrap it into structured HTML markup, which is then sent to the browser."

Introduction to Templates - Theme Handbook

Fake Template Example


						<article>
							<header>
								<h2><!-- Output title --></h2>
								<span><!-- Output author name --></span>
							</header>
							<section>
								<!-- Output dynamic content here -->
							</section>
						</article>

DRY Principle

The "Don't Repeat Yourself" principle is much easier to follow with a templating system.

For example, you could have a website with thousands of blog posts that uses one template file to load all of those webpages.

WordPress Templates

WordPress themes are a collection of template files that work together to produce a website.

WordPress has a very specific templating system that allows for easy control over the visual presentation of different content types.

Templates vs Template Parts

WordPress Block Themes have a templates folder and a parts folder.

  • Templates: Top level templates that include everything inside the body element.
  • Template Parts: Partial templates to be used inside of the top level templates.

WordPress Template Markup

Templates in block themes are written in HTML with "block markup". Block markup is HTML comments with data that WordPress parses into dynamic content.


						<!-- wp:template-part {"slug":"header","tagName":"header"} /-->

							<!-- wp:group {"tagName":"main","layout":{"type":"constrained"}} -->
							<main class="wp-block-group">

								<!-- wp:template-part {"slug":"loop","align":"full"} /-->

							</main>
							<!-- /wp:group -->

						<!-- wp:template-part {"slug":"footer","tagName":"footer"} /-->

Common Template Files

  • index.html (required)
  • page.html

How does WordPress know which template to use?

WordPress Template Selection

WordPress uses the URL to determine which template to use to display a webpage.

  1. A visitor goes to example.com/hello-world/
  2. WordPress searches the database for "hello-world"
  3. It finds out "hello-world" is a Post
  4. It looks for a template to render a single Post

Template Hierarchy

When WordPress gets to step four, it now looks at the theme to find an appropriate template.

WordPress IDs

WordPress IDs

Everything created or uploaded to WordPress is assigned a unique ID.

Examples: a Page, a Blog post, an image.

This ID is stored in the WordPress database and will never change or be reused, even if that item is deleted.

Sidebar Wrapper Element

Register Template Parts

Add the following to theme.json:


						"templateParts": [
							{
								"area": "header",
								"name": "header",
								"title": "Header"
							},
							{
								"area": "footer",
								"name": "footer",
								"title": "Footer"
							},
							{
								"area": "uncategorized",
								"name": "sidebar",
								"title": "Sidebar"
							}
						],
